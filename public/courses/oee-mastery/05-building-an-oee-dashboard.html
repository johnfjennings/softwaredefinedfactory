Building an OEE Dashboard

From raw data to real-time visibility

Software Defined Factory

Why a Dashboard?

A spreadsheet tells you what happened yesterday.

A dashboard tells you what's happening right now.

  • Real-time OEE visibility for operators and management
  • Instant identification of active problems
  • Trend analysis to verify improvement efforts
  • Shift handover information at a glance

Dashboard Layout: The Big Picture

An effective OEE dashboard has four zones:

  1. OEE Score — Large, color-coded, impossible to miss
  2. Sub-Metrics — Availability, Performance, Quality gauges
  3. Loss Breakdown — Pareto chart of current losses
  4. Trend View — OEE over time (shift, day, week)

Zone 1: The OEE Score

The headline number. Design guidelines:

  • Large font — visible from 10+ meters away on a shop floor display
  • Color-coded: Green 85%+ | Yellow 60-84% | Red <60%
  • Update frequency: every 5-15 minutes (not every second)
  • Show current shift OEE, not just the last hour

Zone 2: Sub-Metric Gauges

Show each factor with its target line so operators can immediately see which one is dragging OEE down.

Zone 3: Loss Pareto

A horizontal bar chart sorted by impact:

Top losses this shift:

  • Changeover — 35 min (42%)
  • Minor stops — 22 min (26%)
  • Speed loss — 15 min (18%)
  • Breakdown — 8 min (10%)
  • Defects — 3 min (4%)

The Pareto chart answers: "Where should we focus right now?"

Zone 4: Trend View

Show OEE over time at multiple levels:

  • Hourly — Within a shift, spot when losses occur
  • Daily — Compare day-to-day consistency
  • Weekly — See the improvement trajectory

Add a target line (e.g., 75%) and a trend line to show direction.

Operators need to see: "Are we getting better?"

Comparison Views

Beyond the single-line dashboard, add views to compare:

Comparison What It Reveals
Line vs. Line Which equipment needs the most attention?
Shift vs. Shift Are there training or staffing gaps?
Product vs. Product Which products are hardest to run efficiently?
Week vs. Week Is the improvement trend real or noise?

Dashboard Design Rules

  1. 5-second rule: Anyone should understand the status in 5 seconds
  2. Color = action: Green = keep going, Yellow = monitor, Red = intervene
  3. Less is more: Max 6-8 KPIs per screen. Add detail on drill-down pages.
  4. Match the audience: Operators need real-time; managers need trends
  5. Show targets: A number without a target is just a number

Shop Floor Display Tips

  • Use a large TV (42"+) mounted where operators can see it
  • Auto-refresh — no manual interaction needed
  • High contrast colors for factory lighting conditions
  • Include current downtime timer if equipment is stopped
  • Show shift target vs. actual production count

Cost: A Raspberry Pi ($50) + TV ($200) + free dashboard tool = shop floor OEE display for under $300.

Key Takeaways

  • A dashboard turns OEE from a historical report into a real-time management tool
  • Four zones: OEE score, sub-metrics, loss Pareto, trend view
  • Start simple — even a Google Sheet on a TV is better than no visibility
  • Design for the audience: operators need action signals, managers need trends
  • The dashboard doesn't improve OEE — the actions people take because of it do
